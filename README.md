# MRDR Report Updater

## Что делает программа

Программа обновляет **существующий Excel-отчёт** данными из Excel-выгрузки и умеет ставить **ежедневный автозапуск через Планировщик заданий Windows**.

Логика такая:

- читает лист `products` из файла выгрузки;
- ищет в выгрузке колонки:
  - `Артикул (MRDR) продукта`
  - `Дата создания`
  - `Значения Прогресса`
- оставляет только строки:
  - где артикул не пустой;
  - где дата создания распознана;
  - где дата создания **не раньше 01.01.2026**;
- убирает дубли по артикулу, оставляя **последнюю запись**;
- открывает **уже существующий** Excel-отчёт;
- находит или создаёт колонку прогресса для текущей выгрузки по **дате изменения файла выгрузки**;
- для новых артикулов добавляет новую строку;
- для существующих артикулов записывает новое значение прогресса в соответствующую колонку;
- выравнивает содержимое ячеек по центру и подстраивает ширину колонок.

## Как работает отчёт

### Входной файл выгрузки

Программа ожидает Excel-файл, в котором есть лист:

- `products`

На этом листе должны быть колонки, содержащие в названии:

- `артикул` и `mrdr`
- `дата` и `создания`
- `прогресс`

Поиск колонок идёт **не по точному совпадению**, а по содержимому текста в названии.  
Это значит, что колонка может называться не строго один в один, но должна содержать нужные слова.

### Фильтрация данных

После чтения выгрузки применяются такие правила:

1. Артикул не должен быть пустым.
2. Дата создания должна успешно преобразоваться в дату.
3. Дата создания должна быть `>= 2026-01-01`.
4. Дубли по артикулу удаляются, остаётся последняя строка.

### Обновление Excel-отчёта

Программа работает с листом:

- `Лист1`

Используемая структура отчёта:

- строка `1` — дата/время выгрузки для колонок прогресса;
- строка `2` — заголовки;
- строка `3` и ниже — данные.

Колонки:

- колонка `A` — `Артикул (MRDR) продукта`
- колонка `B` — `Дата создания`
- колонка `C` и далее — значения прогресса по датам выгрузки

Дата колонки прогресса берётся из **времени последнего изменения файла выгрузки**  
и записывается в формате:

`dd.mm.yyyy`
`hh:mm`

То есть в Excel это выглядит как дата и время на двух строках одной ячейки.

Если колонка с такой датой уже существует, программа пишет в неё.  
Если нет — создаёт новую.

## Интерфейс программы

В окне доступны поля и кнопки:

### Поля

- **Путь к выгрузке** — Excel-файл источника

- **Сохранить настройки** — сохраняет пути и время в `settings.json`
- **Установить автозапуск** — создаёт ежедневную задачу Windows
- **Обновить отчёт** — запускает обновление вручную

## Автозапуск

Автозапуск создаётся через команду Windows:

- `schtasks`

Имя задачи:

- `MRDR_Report_Updater_Daily`

При установке автозапуска программа:

1. создаёт файл `run_update.bat`;
2. регистрирует ежедневную задачу в Планировщике;
3. в указанное время запускает программу в режиме:
   - `--autoupdate`

### Что делает тихий режим

В тихом режиме программа:

- читает `settings.json`;
- берёт из него пути к выгрузке и отчёту;
- если файлы существуют — обновляет отчёт;
- если что-то не так, просто завершает работу без окон ошибок.

Это сделано специально, чтобы автозапуск не мешал пользователю всплывающими сообщениями.

Пример:

{

## Технические детали

### 1. Где программа хранит свои файлы

Функция `app_dir()` определяет рабочую папку так:

- если программа запущена как `.exe` — папка, где лежит `.exe`;
- если как `.py` — папка самого скрипта.

Из-за этого:

- `settings.json`
- `run_update.bat`

будут создаваться рядом с исполняемым файлом.

### 2. Нормализация текста

Для поиска колонок и обработки значений используются функции:

- `normalize_text()` — приводит текст к нижнему регистру, убирает переносы строк и лишние пробелы;
- `normalize_article()` — очищает артикул от неразрывных пробелов и значений вроде `nan`.

### 3. Дата выгрузки

Дата колонки прогресса берётся не из содержимого Excel, а из:

- `os.path.getmtime(export_path)`

То есть это **дата последнего изменения файла на диске**.

### 4. Добавление новых строк

Новая строка ищется через первую пустую ячейку в колонке артикула, начиная с 3-й строки.

### 5. Форматирование

После обновления программа:

- выравнивает всё по центру по горизонтали и вертикали;
- включает `wrap_text` для 1-й строки и строки заголовков;
- ставит высоту:
  - 1-я строка — `34`
  - 2-я строка — `34`
  - строки данных — `22`
- задаёт ширину колонок:
  - артикул — `18`
  - дата создания — `14`
  - прогресс — `12`

## Описание основных функций

### `read_export_file(export_path)`
Читает выгрузку с листа `products`, ищет нужные колонки, фильтрует строки, удаляет дубли и возвращает `DataFrame`.

### `write_report(report_path, data, export_modified_dt)`
Открывает отчёт, создаёт или находит колонку прогресса, обновляет строки по артикулам и сохраняет файл.

### `update_report(export_path, report_path)`
Основной сценарий обновления. Проверяет пути, читает выгрузку, записывает результат в отчёт и возвращает статистику.

### `install_task(run_time)`
Создаёт ежедневную задачу Windows через `schtasks`.

### `run_silent_update()`
Тихий запуск для автозадачи без UI и без всплывающих ошибок.

### `App`
GUI-приложение на `tkinter`.

## Возможные ошибки

### `Файл выгрузки не найден`
Путь к файлу выгрузки не указан или файл был перемещён/удалён.

### `Файл отчета не найден. Нужно выбрать уже существующий отчет`
Программа **не создаёт новый отчёт с нуля** в пользовательском сценарии.  
Нужно выбрать уже существующий файл отчёта.

### `Не найдены нужные колонки в выгрузке`
На листе `products` не найдены нужные заголовки.  
Нужно проверить названия колонок.

### `Время должно быть в формате ЧЧ:ММ`
Неверно заполнено поле времени автозапуска.

### Ошибка создания задачи в планировщике
Обычно это значит одно из двух:

- нет прав на создание задачи;
- система не дала выполнить `schtasks`.

## Ограничения текущей реализации

1. Программа жёстко ожидает лист выгрузки с именем `products`.
2. Программа жёстко пишет в лист отчёта `Лист1`.
3. Минимальная дата создания зашита в коде:
   - `2026-01-01`
4. Обновление идёт только по одному ключу:
   - артикул
5. При тихом запуске ошибки подавляются без логирования.
6. Программа не проверяет тип значения прогресса и пишет его как есть.
